# 移动端 Web DApp 认证路径总览

本文用于在“移动端没有浏览器钱包插件”的前提下，快速选择认证路径。

## 路径清单

- [钱包 App 连接认证（WalletConnect / Deep Link）](./移动端-钱包App连接认证.md)
- [嵌入式钱包认证（托管 / MPC）](./移动端-嵌入式钱包认证.md)
- [SSO / WebAuthn 认证（JWT）](./移动端-SSO与WebAuthn认证.md)
- [中心化 UCAN 发行](./移动端-中心化UCAN发行.md)
- [PC 首次绑定 + 移动端长期会话](./移动端-PC首次绑定长期会话.md)
- [传统账号认证（手机号 / 邮箱 + 2FA）](./移动端-传统账号认证.md)

## 快速选择

- 需要保留链上身份并尽量沿用钱包体验：优先“钱包 App 连接认证”，其次“嵌入式钱包认证”。
- 追求移动端登录体验和接入速度：优先“SSO / WebAuthn 认证”或“传统账号认证”。
- 需要 UCAN 令牌形态但接受中心化信任：使用“中心化 UCAN 发行”。
- 允许用户在 PC 完成一次钱包认证：可选“PC 首次绑定 + 移动端长期会话”。

## 决策问题

1. 业务是否必须保留链上身份来源？
2. 能否接受用户安装钱包 App 或接入钱包 SDK？
3. 是否接受中心化身份与中心化授权服务？
4. 是否允许“PC 首次绑定、移动端长期免签”流程？
