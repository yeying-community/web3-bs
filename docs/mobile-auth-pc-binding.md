# 方案 E：PC 首次绑定 + 移动端长期会话

## 1. 适用场景

- 允许用户在 PC 完成一次钱包登录
- 移动端希望长期免钱包

## 2. 核心思路

PC 完成 UCAN 或 SIWE 登录后，为移动端签发长期会话（JWT/委任 token）。

## 3. 详细落地流程

1) PC 端钱包登录（UCAN/SIWE）
2) 后端生成“移动端绑定票据”
3) 通过一次性链接/二维码/邮件发送到手机
4) 手机打开链接绑定设备，换取长期 JWT
5) 移动端使用 JWT 访问多后端 + WebDAV

## 4. 后端改造清单

- 新增绑定票据接口与设备绑定逻辑
- 绑定票据短期有效、一次性
- 设备解绑与吊销机制

## 5. 优缺点

- **优点**：保留链上身份来源
- **缺点**：多一步绑定流程；需要安全设计

## 6. 结论

适合“允许 PC 首次认证”的业务模型。
